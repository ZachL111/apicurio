# Field Notes

`apicurio` is easiest to review by starting with the fixture, not the prose.

The domain cases cover `contract drift`, `schema width`, `breaking risk`, and `consumer reach`. They sit beside the smaller starter fixture so the project has both a compact scoring check and a domain-flavored review check.

The widest spread is between `consumer reach` and `contract drift`, so those are the first two cases I would preserve during a refactor.

The language-specific addition keeps the review model typed and compiled before execution.
